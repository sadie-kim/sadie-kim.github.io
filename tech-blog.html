Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'. (But not those ones).

Describe what the DOM is and an example of how you might interact with it.

DOM is a web interface that allows us to access documents and elements so we can use them. DOM can be used to make functions work with JavaScript or add styling using CSS.

Explain the difference between accessing data from arrays and objects.

An array and an object are both used to store more than one values in a variable. However, array’s values can be accessed by referring to an index number. Where object’s values can be accessed by providing properties to access the stored value in an object.

Explain what functions are and why they are useful.

Functions are used in JavaScript to perform a particular task. When programming, often same tasks need to be repeated. When a function to perform the repeated task is defined, it simplifies the code.

Debugging and Problem Solving in Tech and Beyond

May 12

Tell your non-tech friend a story about a time you:

Got blocked on a simple problem
I was writing functions and it was showing error because in some places I used ‘drink’ and in some places I used ‘drinks’. I could not figure out what was wrong with my code. I didn’t think my issue would be as simple as that. I spend a long time trying to figure what the issue was and at the end I ended up sending my code to the facilitator and learnt what my issue was. I felt frustrated until I figured out the issue as it felt like I was stuck on a wall. The moment I realised that the issue was something very minor, I could finally relate to those programmer memes I saw on the internet. Now I know to check for singular and plural in my words to look for errors next time. I feel like as I experience more of these errors, it will be easier to spot the errors as I know what to look for.
Solved a problem in an elegant way.
I was able to resolve a few issues myself by googling and watching YouTube videos.

Reflect on how confident you feel using each of these problem-solving techniques/processes:

Pseudocode
I haven’t used Pseudocoding much as I haven’t created many codes from scratch, but I will be using this technique a lot throughout the bootcamp.
Trying something
Trying something, aka trial and error is the technique I’ve been using a lot in my coding journey so far. I think this is a good way to learn to code.
Rubber ducky method
This is a method I already use a lot in my personal life, similar to talking to myself.
Reading error messages
I have been using this since I learnt to use the DevOpsTools. This is a powerful problem-solving method as it indicates where exactly is the error.
Console.logging
I have definitely been using console.log a lot as by using this I can check where the code is going.
Googling
I have been using google search to learn and resolve my issue. I feel that I have room to improve in using the google search technique. The more I learn about coding, the better words and terms I will know to find specific information quickly.
Asking your peers for help
I did not feel comfortable doing that because I do not know other students personally yet, and I was always behind the course curriculum and the peers were already discussing the topics ahead of me. But when I was really stuck, I reached out to a few to ask for help.
Asking coaches for help
I did not feel comfortable doing this in the beginning as the coaches seemed pretty busy. But once Joseph let me know that he is happy to help as much as I needed, I stepped up to ask for help with my progression.
Improving your process with reflection
Reflection has been very helpful as by writing down my feelings and thoughts, it helps me to understand myself better and room for improvement.
